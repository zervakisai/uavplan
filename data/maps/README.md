# data/maps/

Offline-generated map tile files (.npz format) for OSM-realistic scenarios.

## Contents

This directory stores preprocessed map data generated by the `tools/osm_pipeline/`
scripts. Each `.npz` file contains numpy arrays representing a geographic tile:

- `heightmap` (float32, [H, W]) -- building heights in meters, rasterized from OSM
- `road_mask` (bool, [H, W]) -- drivable road cells
- `building_mask` (bool, [H, W]) -- building footprint cells
- `metadata` (dict) -- bounding box, CRS, resolution, OSM query timestamp

## Usage

1. Generate tiles using the offline pipeline:
   ```bash
   python -m tools.osm_pipeline.fetch --bbox 37.97,23.72,37.98,23.73 --out data/maps/
   ```

2. Reference tile files from scenario YAML configs:
   ```yaml
   extra:
     map_source: "osm"
     tile_file: "athens_37.97_23.72.npz"
   ```

## Important

- These files are NOT committed to git (see .gitignore)
- Typical tile size: 500x500 grid at 1m resolution ~ 2MB per tile
- Regenerate locally using the pipeline tools
