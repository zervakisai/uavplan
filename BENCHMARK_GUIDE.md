# UAVBench End-to-End Guide

This is the full operational guide for understanding, running, validating, and demoing UAVBench.

## 1. What The Benchmark Does

UAVBench is a deterministic 2D operational UAV navigation benchmark with:

- 34 scenario IDs (fixed catalog)
- two paper tracks:
  - `static` control track
  - `dynamic` forced-replanning stress track
- planner-agnostic forced path interdictions
- causal dynamic interactions (fire, NFZ, traffic, population risk)
- feasibility guardrails with formal runtime logs
- paper export pipeline (CSV + LaTeX + reproducibility manifest + ablations)

## 2. End-to-End Pipeline

1. Scenario YAML loads into `ScenarioConfig`.
2. `UrbanEnv` builds static map + dynamic layers.
3. Initial plan is generated by selected planner.
4. In dynamic track, deterministic interdictions trigger 1-2 corridor breaks.
5. Planners replan under fairness constraints (same snapshot/budget/cadence).
6. Guardrail enforces post-update feasibility.
7. Metrics and events are exported for publication.

## 4. Tracks And Semantics

- `paper_track=static`
  - expected near-perfect baseline success (`astar`, `theta_star`)
  - no forced breaks
- `paper_track=dynamic`
  - deterministic path breaks at `event_t1`, `event_t2`
  - coupled dynamic pressure from blocking/risk layers

Loader fallback when `paper_track` is omitted:

- `naturalistic -> static`
- `stress_test -> dynamic`

## 5. World Model

### 5.1 Blocking layers

- static buildings/NFZ
- forced interdiction mask
- traffic closure mask
- dynamic NFZ mask
- optional fire/traffic/intruder blocking behavior

### 5.2 Risk (non-blocking) layers

- population/crowd risk map
- adversarial UAV near-miss risk
- smoke contribution
- fused `risk_cost_map`

### 5.3 Causal interactions

- fire -> NFZ expansion pressure
- fire -> road closures
- congestion -> risk amplification

## 6. Fair Evaluation Protocol

### 6.1 Invariants

1. Same snapshot per planning call.
2. Same plan budget per call.
3. Same replanning cadence and max replans.
4. Same collision checker and discretization.
5. Same seed -> same dynamics/events.

### 6.2 Runtime fields

- `plan_budget_static_ms`
- `plan_budget_dynamic_ms`
- `replan_every_steps`
- `max_replans_per_episode`

### 6.3 Planner-agnostic interdiction fairness

- reference corridor is computed once at reset using:
  - `interdiction_reference_planner: astar|theta_star|jps`
- breakpoints are scheduled on that reference corridor (not on per-planner path)
- fairness metric:
  - `interdiction_hit_rate_reference`
  - `interdiction_hit_rate_reference_var_across_planners`

### 6.4 Replanning trigger contract (shared)

Adaptive replanning can trigger only via:

1. path invalidation (upcoming corridor intersects dynamic blockers)
2. forced event activation
3. cadence trigger (`replan_every_steps`)

Global limit:
- `max_replans_per_episode` identical for all planners.

Logged counters:
- `replan_trigger_path_invalidation_count`
- `replan_trigger_forced_event_count`
- `replan_trigger_cadence_count`
- `replan_trigger_stuck_fallback_count`

### 6.5 Formal feasibility guarantee

Guardrail sequence after each dynamic update:

1. reachability check current->goal
2. relax forced blocks
3. relax NFZ/closures
4. emergency corridor fallback
5. hard deconfliction fallback if needed

Logged proof fields:

- `reachability_failed_before_relax`
- `relaxation_applied` (rate/radius/closure deltas)
- `corridor_fallback_used`
- `feasible_after_guardrail`
- `guardrail_activation_rate`
- `corridor_fallback_rate`
- `relaxation_magnitude`

### 6.6 Dual-use operational constraints

- dynamic restricted zones
- safety/deconfliction buffers
- emergency managed corridor
- mission pressure under safety-critical conditions
