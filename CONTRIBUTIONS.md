# Contributions — UAVBench

## Scientific Gap (one sentence)

> No existing UAV planning benchmark simultaneously guarantees
> **deterministic stress instrumentation**, **formal feasibility**,
> and **fair cross-paradigm evaluation** under causally coupled
> dynamic hazards.

## Related Work Positioning

The literature positioning matrix (`literature_positioning_matrix.csv`,
generated by `scripts/paper_best_paper_validation.py`) compares UAVBench
against AirSim, CARLA, common grid RL benchmarks, sampling-based UAV
simulators, and dynamic path-planning testbeds across seven dimensions:

| Dimension | UAVBench | Others |
|-----------|----------|--------|
| Dynamic hazards | ✅ Causal fire/traffic/NFZ | Scenario-dependent |
| Causal coupling | ✅ Fire→NFZ, Fire→road closure, congestion→risk | Typically absent or partial |
| Deterministic stress instrumentation | ✅ Planner-agnostic reference-path interdictions | Not a benchmark primitive |
| Feasibility guarantee | ✅ 4-depth guardrail with runtime proof fields | Not a benchmark contract |
| Fair cross-planner protocol | ✅ 5 invariants (snapshot, budget, cadence, collision, seed) | Partial at best |
| Dual-use operational semantics | ✅ Civil + defense missions | Application-dependent |
| Statistical seed sweeps | ✅ CI, Cohen's d, Mann-Whitney, Kendall τ | Possible but not standard |

**UAVBench is the only benchmark that satisfies all seven dimensions as
first-class contracts.**

## Contributions

### C1 — Benchmark Architecture (Falsifiable)

**Claim:** UAVBench provides a modular, scenario-driven benchmark that
supports 3 mission types × 3 difficulty levels × N seeds with 6 planners
from 4 algorithmic paradigms.

**Falsification criterion:** Falsified if any scenario fails to load, any
planner cannot be substituted without code changes to the runner, or the
3×3 mission–difficulty matrix is incomplete.

**Evidence:**
- `src/uavbench/scenarios/configs/` — 9 YAML scenario configs
- `src/uavbench/planners/` — 6 planners registered in `PLANNERS` dict
- `src/uavbench/benchmark/runner.py` — BenchmarkRunner loops scenarios × planners × seeds
- `tests/test_sanity.py` — registry, loader, and planner tests (245+ total)

### C2 — Fair Evaluation Protocol with Formal Guarantees (Falsifiable)

**Claim:** All planners receive identical snapshots, time budgets, replanning
cadence, and collision checking per evaluation step. The feasibility guardrail
ensures that no episode becomes permanently infeasible due to dynamic
obstacles.

**Falsification criterion:** Falsified if:
- Two planners receive different grid snapshots at the same step and seed
  (determinism test)
- Time budget differs across planners in the same scenario (budget audit)
- Any default-variant episode ends with `feasible_after_guardrail = false`
  (feasibility proof)
- Removing the guardrail does NOT increase infeasibility (no_guardrail ablation)

**Evidence:**
- `src/uavbench/benchmark/fairness_audit.py` — 4-check audit (seed
  reproducibility, snapshot equality, time budget, interdiction hit rate)
- `src/uavbench/benchmark/solvability.py` — disjoint-path certificate
- `docs/PAPER_PROTOCOL.md` — 5 invariants, guardrail sequence, proof fields
- `scripts/paper_best_paper_validation.py` → `_fairness_audit()` function
- `tests/test_vv_contracts.py` — dedicated V&V acceptance tests

### C3 — Causal Dynamic Interactions Create Measurable Behavioral Separation (Falsifiable)

**Claim:** Causally coupled dynamic hazards (fire → NFZ expansion,
fire → road closures, congestion → risk amplification) produce statistically
significant performance differences between static and adaptive planners,
with Cohen's d ≥ 0.5 on at least one key metric.

**Falsification criterion:** Falsified if:
- Cohen's d < 0.5 between static and adaptive planner groups on success rate
  in dynamic scenarios (H1)
- The `no_interactions` ablation variant shows no success-rate delta vs
  default (interaction contribution is zero)
- Fire–traffic feedback rate is 0 in all wildfire scenarios (H6)

**Evidence:**
- `src/uavbench/benchmark/theoretical_validation.py` — 7 hypotheses with
  falsifiability statements, Cohen's d, CI computation
- `scripts/paper_best_paper_validation.py` → stress curves, ablation deltas,
  interaction feedback metrics
- Generated artifacts: `effect_sizes.csv`, `ablation_deltas.csv`,
  `interaction_feedback_metrics.csv`, `paper_theoretical_validation.json`

## Hypothesis Registry

| ID | Claim (short) | Metric | Direction | Falsification |
|----|---------------|--------|-----------|---------------|
| H1 | Adaptive > static in dynamic scenarios | success_rate | greater | Static ≥ 80% success in dynamic |
| H2 | Risk-aware reduces exposure | risk_exposure_integral | less | Risk-aware ≥ non-risk-aware |
| H3 | Distinct replan distributions | total_replans | different | Cohen's d < 0.5 |
| H4 | Path length separates quality | path_length | greater | Normalized range < 0.1 |
| H5 | Guardrail is necessary | feasible_after_guardrail | greater | no_guardrail ≤ 5% increase |
| H6 | Fire-traffic causal feedback | fire_traffic_feedback_rate | greater | Rate = 0 in all wildfire |
| H7 | MPPI reduces replan storms | total_replans | less | MPPI ≥ DWA replans |

Each hypothesis links to a specific metric, comparison groups, effect-size
threshold, and an explicit falsification condition — ensuring every claim
in the paper is empirically disprovable.
